# 行为准则

MnemoNAS 希望成为一个友好、专业、以用户数据安全和长期可靠性为中心的开源项目。参与 Issue、Discussion、Pull Request、代码评审或其他项目沟通时，请遵守本行为准则。

## 我们鼓励的行为

- 尊重不同经验背景、语言、时区和使用场景的参与者
- 用具体事实、日志、复现步骤和技术权衡讨论问题
- 友善地给出反馈，并接受建设性的不同意见
- 聚焦项目目标、用户安全、可维护性和长期协作
- 发现安全问题时按 [Security Policy](SECURITY.md) 私下报告，避免公开披露细节

## 不可接受的行为

- 人身攻击、骚扰、威胁、侮辱或持续打断他人讨论
- 基于身份、地域、语言、经验水平等因素的歧视性言论
- 发布他人的隐私信息、凭据、日志中的敏感数据或未授权数据
- 故意误导维护者、刷屏、重复提交无关内容，或破坏项目协作秩序
- 在公开渠道披露未修复安全漏洞的可利用细节

## 处理方式

维护者可以根据具体情况采取以下措施：

- 提醒参与者调整沟通方式
- 编辑、隐藏或删除违反准则的内容
- 暂时或永久限制参与项目的权限
- 对严重安全、骚扰或滥用行为直接升级处理

## 报告问题

如果你遇到或看到违反本准则的行为，请通过 GitHub Issue、Discussion 或维护者公开资料中提供的联系方式反馈。涉及安全漏洞时，请优先使用 GitHub Private Vulnerability Reporting，并参考 [SECURITY.md](SECURITY.md)。

报告时请尽量包含相关链接、时间、上下文和你希望维护者注意的影响。维护者会在合理范围内保护报告人的隐私。
