# 行为准则

[English](CODE_OF_CONDUCT.md) | 简体中文

本文档定义 MnemoNAS 问题反馈渠道的行为预期，适用于公开 Issues、私密安全报告及维护者处理反馈时的相关互动。

## 预期行为

参与者应：

- 将讨论聚焦于具体项目工作、可复现证据和可维护结果；
- 在技术分歧中保持尊重；
- 将技术审查意见与个人评价分开；
- 为缺陷、使用问题、功能建议、兼容性结果和运维报告提供足够背景；
- 发布日志或截图前移除密码、Token、Cookie、私密 URL、内网敏感地址和个人数据。

## 不可接受行为

以下行为在项目空间中不可接受：

- 骚扰、威胁、人身攻击、侮辱或歧视性言论；
- 持续干扰技术讨论；
- 未经明确授权发布他人私密信息；
- 在公开渠道发布漏洞利用细节、凭据、Token 或敏感部署信息；
- 明知不实仍提交误导性报告、伪造证据或无关推广内容。

## 安全敏感报告

安全漏洞、凭据泄漏、数据丢失风险、路径穿越、公网暴露风险和类似敏感报告应遵循 [安全策略](SECURITY.zh-CN.md)。公开 Issues 不应包含漏洞利用细节或运维密钥。

## 执行

维护者可以编辑、隐藏或移除违反本行为准则的评论、Issues 或其他反馈内容。当相关行为反复干扰项目工作或造成安全风险时，维护者也可以限制或阻止参与。

执行决定应基于项目安全、证据、比例原则，以及维护有效技术协作环境的需要。

## 适用范围

本行为准则适用于 MnemoNAS 问题反馈渠道和相关互动。它不替代 [反馈处理边界](SUPPORT.md) 或 [安全策略](SECURITY.zh-CN.md)。
