# MnemoNAS Flutter 客户端

[English](README.en.md) | 简体中文

本目录包含面向 Android、Linux 和 Windows 的 Flutter 客户端。Android 是首个可用平台目标；Linux 和 Windows runner 当前仅保留共享的跨平台工程边界，尚未完成对应原生主机上的构建和发布验证。

> [!WARNING]
> 客户端仍处于开发阶段，尚未发布任何可用版本或可分发安装包。当前源码和构建产物仅用于开发与验证。

## 当前开发范围

当前源码已接入：

- 服务器地址输入、服务状态验证和设备切换；
- 登录、安全会话存储、访问令牌刷新、强制修改密码、主动修改密码和退出登录；会话更新使用 revision/CAS，退出、切换设备和重新登录会使旧请求失效；
- 设备概览及文件目录浏览；
- 按文件名搜索当前账户可访问的文件和目录；每次最多显示 100 项，切换搜索词会取消旧请求，打开结果前会重新读取目标目录；
- 新建文件夹、上传、下载、打开、重命名、移动、复制和基于服务端删除策略的删除确认；目录响应必须返回与请求一致的规范路径，文件页系统返回会先取消多选，再逐级返回父目录；
- 对具备具体读取权限的版本化普通文件查看版本历史；版本 API 以规范路径建立当前 BLAKE3 内容身份并返回经过严格校验的版本序列，当前版本卡只展示身份和元数据，历史版本可预览或下载，当前文件的打开和下载仍从文件页执行；管理员恢复前会重新核对目录、当前内容身份和所选历史版本，恢复请求不会自动重放，结果无法确认时也不会提示重复提交；
- 前台下载任务使用应用私有持久账本和稳定局部文件，支持暂停、继续及客户端重启后恢复；续传同时校验服务端下载身份、`206`、`Content-Range` 和总大小；
- 前台上传任务使用同一持久账本、任务专属私有载荷和服务端可恢复会话；首次创建会话前先持久化尝试标记，创建响应中断且会话 ID 未知时只按客户端请求 ID 查询原会话，不会再次创建目标快照；客户端按 8 MiB 分块写入，以服务端可靠偏移继续，并通过幂等提交查询终态；原会话缺失或过期时标记为“结果待确认”；
- 传输中心按“进行中”“需要处理”和“最近记录”分组展示任务；暂停或等待处理的任务可继续执行，Android 完成私有下载后可继续选择保存位置，暂停或失败任务必须确认后才能取消并删除本地可恢复进度，“结果待确认”记录也要求先核对目标位置；应用进入后台时，传输中心内的持久任务会保存为暂停或等待重新选择保存位置，临时文件预览、历史版本预览或下载及未完成的 SAF 上传准备会取消并清理局部文件，返回前台后不会自动继续；
- Android 上传选择使用 Storage Access Framework，只向 Dart 返回 URI 与元数据，并按文件逐个流式复制到应用私有目录，不把所选文件完整载入 Java heap；已知大小超过 10 GiB 时会在复制前拒绝，大小未知时原生复制同样执行 10 GiB 硬限制并清理超限局部文件；准备进度单独显示，任务载荷完成 SHA-256 校验并写入持久账本后，临时导入文件即可释放；
- 回收站列表、逐项到期时间、原路径或自定义路径恢复，以及按冻结 ID 集合永久删除；请求终态无法确认时会重新读取回收站，但不会仅凭项目消失推断恢复或永久删除成功，再次明确刷新前暂停后续修改；
- 客户端与服务端版本展示，以及 GitHub Issues 问题反馈入口。
- Android Release、Debug 和 Profile 使用独立应用身份；release APK/AAB 要求签名配置与密钥库位于源码 checkout 外，并拒绝注入签名覆盖、无效私钥别名、无效证书或不匹配的 SHA-256 指纹，CI 使用临时测试密钥验证 APK/AAB 及绕过回归。

以上能力仅表示当前源码中的实现范围，不表示已经形成可用客户端版本。

## 当前缺口

- 全文与相册索引尚未接入，文件名搜索暂不支持游标分页；
- 分享和其余管理功能尚未形成完整客户端流程；
- Android 原生后台传输执行器、通知控制和跨进程任务租约尚未完成；当前可恢复上传与下载都只在前台 Dart 协调器中运行，进入后台时会持久暂停；
- 当前界面文案以简体中文为主，完整多语言支持尚未完成；
- Linux 和 Windows 尚未完成原生构建与运行验证；
- Android 16 真机已覆盖安装、登录、系统返回层级、进程重启会话恢复和 Debug 同签名升级；正式密钥托管、受控 Release 候选包、API 24/33/34 与完整设备/故障矩阵仍未完成。

客户端允许本机或可信局域网使用 HTTP，并显示未加密警告；公网地址必须使用 HTTPS。
