## 已完成的重构

### 客户端基础与交互

- 建立 Android、Linux 和 Windows 共用的 Flutter 工程、主题与组件边界。
- 接入服务器连接、登录、设备概览、文件浏览、文件名搜索、回收站和账户流程。
- 主页与文件页区分初次加载、同目录刷新和刷新失败后的缓存状态；目录数据过期时保留可读内容并明确提示，不再显示为在线新数据。
- 文件修改使用控制器级单飞租约和界面级即时防抖。已确认修改不会因后续目录刷新失败改写为失败，未确认结果会提示先核对目录，目录切换也不会被迟到刷新拉回旧路径。
- 目录响应必须返回规范路径并与请求路径一致；不可信路径会作为无效响应被拒绝，不会进入 Widget 构建。文件页系统返回依次处理多选取消、父目录导航和根路由退出。
- 将传输记录拆分为独立传输中心，按任务状态分组，并区分暂停、重试、选择保存位置、取消并删除和清除记录；破坏性清理与“结果待确认”记录均要求明确确认。
- 应用进入后台时同步认领传输中心内的当前任务；持久下载、上传和 Android 文档导出保留应用私有载荷与可继续状态，临时文件预览、历史版本预览或下载及 SAF 上传准备会取消并清理局部文件，返回前台后不自动恢复。
- 文档只保留 GitHub Issues 问题反馈入口；README 和开发阶段记录明确说明尚无可用版本。

### 会话与破坏性操作安全

- 会话记录使用 revision/CAS 和原子清除，避免不同 refresh token 代际混用。
- 服务器切换、账号切换、退出和迟到响应使用 context epoch 隔离。
- 删除与回收站流程冻结策略、目标身份和精确 ID；请求终态无法确认时不会推断成功。

### 版本历史与恢复

- 版本历史入口只对具备规范化路径、具体读取权限和版本化标记的普通文件显示，不要求目录条目预先包含内容哈希。版本 API 建立当前 BLAKE3 内容身份，目录条目已携带身份时还会核对一致性；列表严格校验路径、连续版本号、哈希、大小、时间和有界说明文本。
- 当前版本卡只展示身份与元数据；历史版本可以预览或下载，当前文件的打开和下载仍从文件页执行。历史下载保持 Bearer 认证在应用内，并校验精确 strong ETag 与长度；Android 先写入应用私有临时文件，再打开系统保存位置。
- 管理员恢复前重新读取父目录和版本历史，核对初次版本响应建立的当前内容身份与所选历史版本，并复用文件修改单飞租约。恢复请求不自动重放；连接中断、超时、不可解析响应和结构化 `5xx` 均标记为“结果待确认”。
- 已确认恢复但历史刷新失败时，不返回旧历史；当前版本历史弹层会禁用后续恢复，并要求关闭后刷新目录再重新打开。服务端恢复接口尚无当前身份 CAS 参数，因此提交前核对与写请求之间仍存在 TOCTOU。

### 下载恢复

- 服务端当前文件下载增加 `HEAD`、`X-MnemoNAS-Download-Identity` 和 `X-MnemoNAS-If-Download-Identity`。
- 客户端使用应用私有代际账本保存任务，不保存 access token 或 refresh token。
- 下载使用稳定局部文件和可靠偏移。恢复时必须同时满足对象身份、`206`、精确 `Content-Range` 和总大小。
- 客户端重启后先按局部文件实际长度对账：未完成任务恢复为暂停状态，已完整写入 Android 私有载荷的任务等待重新选择目标，可能已发布到桌面目标但无法确认的任务标记为“结果待确认”。
- Android 先把网络内容提交到应用私有载荷，再进入“等待选择保存位置”。保存对话框不会在网络传输前创建空文档；选定目标后使用当前流程的临时写授权，原生复制成功后才标记完成。客户端重启后保留私有载荷，并要求重新选择位置。

### Android 文件选择

- 上传选择改为原生 `ACTION_OPEN_DOCUMENT`，Dart 只接收 URI 和元数据。
- 原生层以 64 KiB 缓冲区逐个复制文件，执行 `flush` 和 `fd.sync`，失败时清理局部文件。
- 上传导入和下载保存都只使用当前前台流程的临时 URI 授权，不持久占用共享授权。可恢复状态保存在应用私有载荷和账本中，该边界同时避免原插件把完整文件和多选批次载入 Java heap。

### Android 发布与设备证据

- Release 保留 `com.mnemonas.app`；Debug 与 Profile 使用独立包名、版本后缀和显示名称。生成 release APK/AAB 前必须提供源码 checkout 外的签名配置与密钥库，并通过私钥别名、证书有效期、Android Debug 证书拒绝和证书 SHA-256 指纹校验。
- 门禁禁止 `android.injected.signing.*` 覆盖，并根据 Gradle 解析后的任务图识别 release 产物任务，任务缩写或排除独立校验任务不能跳过校验。临时 PKCS12 测试密钥覆盖缺失配置、空字段、不可读文件、checkout 内密钥材料、无效别名或口令、证书有效期、Debug 证书、指纹不匹配及两类绕过回归；生成的 APK 与 AAB 均通过实际验签。脚本自动清理测试密钥和 release 产物，CI 运行同一门禁。
- 本轮本地人工验收分别在 API 35 模拟器和 OPPO PLP120 Android 16（API 36）真机执行连接、登录、多选返回、`/e2e-folder-share/subdir → /e2e-folder-share → /` 逐级返回、强制结束进程后的会话恢复，以及同一 Debug 签名下 `versionCode 1 → 2` 覆盖升级后的会话保持。该记录不等同于受控正式签名候选包验收。

### 上传恢复

- 服务端使用所有者隔离的持久上传会话，提供创建与请求 ID 幂等、状态查询、顺序分块、可靠偏移、分块 SHA-256、完整载荷 BLAKE3、条件提交、取消、期限清理和崩溃终态对账。
- 会话存储使用有界双快照状态链、全量保留记录上限、20/100 GiB 暂存字节上限和主机最小可用空间准入；重叠提交合并到同一终态，服务端启动时会对账中断的 `committing` 状态。
- 旧 `POST /api/v1/files/{path}` 继续明确拒绝 `Content-Range`；可恢复客户端只使用独立的 `/api/v1/upload-sessions` 协议。
- 客户端先建立任务专属私有载荷并记录 SHA-256，再把创建尝试状态、会话 ID、服务端可靠偏移和期限写入持久账本。首次创建请求前必须先持久记录尝试；响应丢失且会话 ID 未知时，只按客户端请求 ID 查询已有会话，不重发创建请求。
- 会话缺失或过期时，客户端将任务标记为“结果待确认”，不会以新的目标基线自动重建会话。服务端恢复还必须先通过存储恢复门禁，不根据尚未对账的可见目标提前确认提交。上传成功时只有取得 `committed` 终态后才删除私有载荷；启动恢复会重试清理已确认完成或取消的任务载荷。
- Android 文件元数据已知时，超过 10 GiB 的来源会在复制前被拒绝；大小未知时，原生复制层仍执行 10 GiB 硬限制，并在超限时停止写入和清理局部文件。本地准备使用独立状态显示，避免与网络上传进度混淆。

## 尚未完成

按当前优先级排列：

1. 将前台 JSON 代际账本替换为具备 revision/CAS、持久任务指令和 fencing token 的事务账本，再接入 Android 原生后台执行器、进度通知、暂停与重试动作及系统停止原因。
2. 收紧 Android release 明文流量策略，建立隔离构建、只读正式密钥托管、轮换与恢复流程，实施单调版本发布轨，并对最终 APK/AAB 独立复核摘要、应用身份和证书指纹后完成候选包升级验收；AGP、flavor 或发布任务变化时同步复核签名任务集合。
3. 补齐 API 24/33/34 及更多厂商设备上的大文件、断网、权限撤销、前后台暂停和恢复验收，并复核原生复制失败后由文档提供方保留的空文件或局部文件。
4. 减少 Android SAF 导入到任务私有载荷之间的临时双份空间，并为完整载荷复核提供独立状态。
5. 为桌面下载目标发布补充禁止覆盖的原子原语、持久发布日志和目录同步，并持久化重启后的覆盖决策。
6. 在 Linux 和 Windows 原生主机上完成构建、文件选择、目标发布、路径替换和进程恢复验证。
7. 完成全文与相册索引、搜索游标分页、目录打包下载、分享和其余管理流程；将移动、复制和冲突恢复的绝对路径输入改为可浏览的目标目录选择，并补齐完整多语言与剩余交互一致性检查。
8. 在服务端提供当前身份 CAS 后，收紧版本恢复的提交时竞争窗口。

## 验证原则

每个重构切片先运行最小定向测试，再运行 `make verify-changed` 和 `make client-check`。Android 签名逻辑还必须运行 `make client-android-release-signing-check`；Android 行为变更需要 Kotlin 编译、Debug APK、模拟器流程和真实设备证据。Debug APK 只用于开发验证，不属于可发布产物。
